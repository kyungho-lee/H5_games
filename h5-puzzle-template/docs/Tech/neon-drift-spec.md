## 1. 개요

방향 버튼으로 8×8 그리드 전체를 한 방향으로 밀어, **같은 색·같은 크기** 블록을 합쳐 더 큰 블록으로 승급시키는 슬라이드 병합 퍼즐. 목표는 **단색 지배**(한 색만 남기기)와 **점수 최대화**의 두 축.

장르: 슬라이드 병합 퍼즐 (2048 × 테트리스 하이브리드)
플랫폼: H5 (Playgama / CrazyGames), 모바일·데스크톱 웹.

---

## 2. 핵심 모델 결정 (컨셉 모호성 해소)

컨셉 문서는 "동색 병합"과 "블록 크기 2,4,8 표기"를 동시에 언급해 규칙이 모호했다. 본 스펙은 다음으로 확정한다.

**타일 = `{ color, size }`**

- `color`: 0..N-1 (현재 활성 색상 수, 레벨 진행에 따라 3→6)
- `size`: 2의 거듭제곱 (1, 2, 4, 8, 16, …)
- **병합 조건: 같은 `color` AND 같은 `size`** → 한 단계 위 `size`로 승급 (2048식 크기 계층)
- `color`는 시각 테마 + 스폰 편향 + 승리 판정(단색)에 사용, `size`는 병합 진행 축

이 모델로 컨셉의 두 서술이 모순 없이 양립한다.

## 3. 기능 요구사항

### FR-1 · 그리드 초기화
- 8×8 = 64 셀. 게임 시작 시 활성 색상 수(기본 3색)로 size 1 블록 N개 랜덤 배치(기본 8개).
- 빈 셀은 `null`.

### FR-2 · 슬라이드(밀기)
- 입력: ↑ ↓ ← → (CSS 방향 버튼 + 키보드 화살표 + 터치 스와이프).
- 한 번의 밀기로 **그리드 전체**가 해당 방향으로 압축 이동한다.
- 각 라인(행 또는 열)에서 빈칸을 제거해 한쪽으로 붙인다(compact).

### FR-3 · 병합
- 압축 후 인접한 두 타일이 **같은 color·같은 size**이면 병합 → size ×2, 한 칸으로 합쳐짐.
- **재병합 금지**: 한 번의 밀기 안에서 이미 병합으로 생성된 타일은 같은 밀기에서 다시 병합되지 않는다 (2048 규칙).
- 병합 1건당 점수 = 병합 결과 size.

### FR-4 · 체인(연쇄)
- `chain` = 한 번의 밀기에서 발생한 총 병합 수.
- 배율: chain 1 → ×1, 2 → ×2, 3 → ×3, 4 → ×4, **chain ≥ 5 → ×5 + 와일드카드 보너스 훅**(리워드 광고).
- 밀기 점수 = Σ(병합 결과 size) × chainMultiplier(chain).

### FR-5 · 신규 타일 스폰
- 격자에 **변화가 있었던 경우에만**(moved=true) 빈 셀 1곳에 신규 타일 1개(size 1) 생성.
- 색상 = **현재 그리드 최다색 편향** 가중 랜덤. (RNG는 주입되어 결정론적 — §아키텍처)
- 변화 없는 밀기(moved=false)는 스폰하지 않으며 턴으로 치지 않는다.

### FR-6 · 승리 — 단색 지배
- 격자에 존재하는 distinct `color` 수가 1이 되면 **승리**(레벨 클리어).
- **승리 판정은 신규 타일 스폰 *이전*의 격자에서 수행한다**(applyMove 내부 순서: 병합 → 승리 판정 → 미승리 시에만 스폰). 스폰 후 판정하면 비주류색 스폰이 영구히 승리를 막아 소프트락이 발생한다.
- **단색 임박 클램프**: 격자의 distinct color 수가 ≤ 2이면 신규 타일은 **주류색(최다색)으로 고정 스폰**한다. 이로써 단색 수렴이 실제로 도달 가능해진다(편향만으로는 비주류색이 끼어들어 영구 미승리 가능).
- 승리 시 클리어 연출 → 다음 레벨(색상 수 +1, 최대 6).

### FR-7 · 패배 — 게임 오버
- 빈칸이 0이고 **어느 방향으로도 이동/병합이 불가능**하면(canMove == false) 게임 오버.
- 빈칸이 0이어도 인접 병합이 가능하면 게임 오버가 아니다.

### FR-8 · Undo
- 직전 격자 스냅샷 1단계 보관. **기본 1회** Undo 제공.
- 스냅샷 없음 또는 이미 사용 시 버튼 비활성화.

### FR-9 · 난이도 곡선
- 난이도는 **수식 기반 `difficulty(level)` 순수 함수**로 산출(아키텍처 §3.3). 주 축은 활성 색상 수(3→6, max 6), 부수 팩터로 스폰 편향·시작 블록 수·단색 클램프 임계를 레벨에 따라 조절. 자매 프로젝트 `DIFFICULTIES` 테이블은 보드 구조가 달라 포팅하지 않음.
- 레벨업 전환 시 인터스티셜 광고: `SG.CG.requestMidgameAd()` 호출(내부적으로 인터스티셜 처리).

## 4. 비기능 요구사항

| ID | 항목 | 요구 |
|----|------|------|
| NFR-1 | 프레임 | 슬라이드 트윈 60fps, 라인당 50–80ms |
| NFR-2 | 입력 응답 | 트윈 진행 중 입력 잠금(1푸시=1애니), 큐잉 없음 |
| NFR-3 | 결정론 | 게임 코어는 `Date.now()`/`Math.random()` 미사용, RNG 주입 → 단위 테스트 가능 |
| NFR-4 | 의존성 | 신규 외부 런타임 의존성 0. 테스트는 `node:test` 내장 러너 |
| NFR-5 | 무수정 재사용 | `playgama.js` `firebase.js` `notify.js` `sound.js` 수정 없이 호출만 |
| NFR-6 | 배포 | `src/` 플랫 구조 zip → Playgama QA Tool 업로드 가능 |

## 5. UI / UX

### 5.1 화면 구성
```
┌──────────────────────────┐
│  NEON DRIFT   LV.3       │
│  SCORE: 8,240  CHAIN: ×2 │
├──────────────────────────┤
│   ┌──────────────┐       │
│   │  8×8 GRID    │       │  ← Canvas (슬라이드 트윈)
│   └──────────────┘       │
├──────────────────────────┤
│      [↑]      [↩ UNDO]   │
│  [←] [●] [→]            │  ← 네온 글로우 CSS 버튼
│      [↓]                 │
└──────────────────────────┘
```

### 5.2 연출
- 슬라이드: Canvas lerp, 타일 start→end 좌표 트윈.
- 병합 순간: `ParticleSystem` 파티클 폭발 + `FloatText` 점수 팝업.
- 체인 달성: 화면 전체 글로우 플래시.
- 사운드: `sound.js`의 `playRemove(size)` / `playCombo(chain)` / `playClear` / `playGameOver` 재사용.

### 5.3 디자인 토큰
- [docs/design-tokens.md](../design-tokens.md)의 `--accent`(cyan), `--warn`(red), `--c0..--c5`(타일 색), `Rajdhani`/`Share Tech Mono` 사용.

### 5.4 Dev 모드 검증 (`?dev` / `localStorage.sg_dev`)
- **타일 size 숫자 표시**: 각 타일 중앙에 size 값(1,2,4,8…) + 선택적 color index를 `--textdim`으로 오버레이 → 병합/슬라이드를 화면에서 시각 검증.
- Dev 바 버튼:
  - `격자 덤프`: `console.table`로 `{color,size}` 매트릭스 출력
  - `스폰 고정`: RNG seed 고정 토글 → 재현 가능한 시퀀스
  - `강제 방향`: ↑↓←→ 1스텝 실행 + 직전/직후 grid diff 로그
  - `승리 임박`: 단색 직전 격자 주입 → WIN 판정 검증
- 자동 단위 테스트(§아키텍처) + dev 숫자 표시 = **이중 검증**.

## 7. 게임 생애주기 (상태 머신)

```
BOOT → READY(START 오버레이) → PLAYING → [WIN | GAMEOVER]

PLAYING:
  - 매 밀기: applyMove → 트윈 → 스폰 → 승/패 판정
  - 레벨업 시: requestMidgameAd() → 색상 수 +1
WIN(레벨 클리어): 클리어 연출 → 다음 레벨 PLAYING
GAMEOVER:
  - 리워드 광고(블록 2개 제거) 제안 → 수락 시 PLAYING 복귀
  - 거절 시: 점수 제출(SG.FB / 리더보드) → GAMEOVER 오버레이
```

## 10. 미해결/후속

- 스폰 색상 편향 가중치 구체 수치(최다색 가중 배수)는 프로토타입에서 플레이테스트로 튜닝.
- chain≥5 와일드카드의 정확한 병합 규칙(임의 색·임의 size 모두? size만?)은 구현 시 1차 결정 후 검증.
